9:T1077,<h2>1. 모바일 앱 분석 환경 구축</h2>
<p>| 용도       | 이름                 | 설명                         |
| ---------- | -------------------- | ---------------------------- |
| 에뮬레이터 | Nox App Player       | 구글 검색 후 다운로드 &#x26; 설치 |
| 디컴파일러 | jadx                 | GitHub 최신 Release 다운로드 |
| 디컴파일러 | apkstudio            | APK 분석용 GUI               |
| APK 변환   | apkeasytool portable | apktool 기반 CLI 툴          |
| 자바       | Adoptium OpenJDK     | adoptium 검색 후 설치        |
| 파이썬     | Python 3.x           | python download              |
| 디버깅     | adb                  | ADB &#x26; Fastboot               |</p>

<h3>기타 세팅</h3>
<h4>• Rooting / VT-x</h4>
<ul>
<li>Hyper-V 끄기
<ul>
<li>Windows 기능 → Hyper-V 전체 OFF</li>
<li>Windows Sandbox OFF</li>
</ul>
</li>
<li>VT-x 활성 여부 확인:
<ul>
<li>https://leomoon.com/downloads/desktop-apps/leomoon-cpu-v/</li>
</ul>
</li>
</ul>
<hr>
<h2>2. Nox 최초 세팅</h2>
<ol>
<li>실행 중인 에뮬레이터가 64bit인지 확인 (멀티 실행기 Ctrl+6)</li>
<li>기존 에뮬레이터 삭제 후 Android 9 64bit 추가</li>
<li>톱니바퀴(설정)</li>
<li>성능 → 해상도: 스마트폰(540x960)</li>
<li>일반 → ROOT 켜기</li>
</ol>
<hr>
<h2>3. ADB(Android Debug Bridge)</h2>
<h3>개념</h3>
<p>PC ↔ 스마트폰을 USB로 연결해 제어/파일관리하는 도구</p>
<h3>설치 위치 Path 등록</h3>
<ol>
<li>설정 → 시스템 정보 → 고급 시스템 → 환경 변수</li>
<li>Path → 새로 만들기</li>
<li>Nox/bin 경로 추가</li>
<li>CMD 재실행 필수</li>
</ol>
<hr>
<h3>USB 디버깅 활성화</h3>
<ul>
<li>설정 → 시스템 → 정보 → <strong>빌드번호 7회 탭</strong></li>
<li>개발자 옵션 활성화</li>
<li>개발자 옵션 → USB 디버깅 켜기</li>
</ul>
<hr>
<h2>4. ADB 명령어</h2>
<p>| 명령어      | 설명                   |
| ----------- | ---------------------- |
| adb devices | 연결된 기기 확인       |
| adb shell   | 스마트폰 OS Shell 접속 |
| adb install | APK 설치               |
| adb pull    | 폰 → PC 파일 추출      |
| adb push    | PC → 폰 파일 전송      |</p>
<hr>
<h3>설치된 앱 패키지 확인</h3>
<pre><code>adb shell
pm list packages -f | grep 검색어
</code></pre>
<hr>
<h2>5. APK 설치 (수동/ADB)</h2>
<h3>스마트폰 수동 설치</h3>
<ol>
<li>다운로드 폴더에서 APK 실행</li>
<li>"알 수 없는 개발자" → 설치 허용</li>
</ol>
<h3>ADB 설치</h3>
<pre><code>adb install sample.apk
</code></pre>

<h2>6. apktool 사용법</h2>
<pre><code>apktool d 파일.apk     # 디컴파일
apktool b 폴더명       # 리컴파일
</code></pre>
<p>예시 패키지 구조:</p>
<pre><code>com.scottyab.sample.rootbeer
→ com/scottyab/sample/rootbeer
</code></pre>

<h2>7. Activity 분석</h2>
<p>예시:</p>
<pre><code>android:name="kr.co.eqst.aos.app008.SplashActivity"
</code></pre>
<p>경로 구조:</p>
<pre><code>kr → co → eqst → aos → app008 → SplashActivity.java
</code></pre>
<h3>exported = true/false</h3>
<ul>
<li><code>true</code> : 외부에서 Activity 호출 가능</li>
<li><code>false</code>: 앱 내부에서만 사용 (보안에 중요함)</li>
</ul>
<p>예)</p>
<pre><code>kr.co.shop → kr.co.kakao.pay.payactivity (exported=false)
</code></pre>

<h2>9. 앱 종류 비교</h2>
<h3>▲ Web App / Mobile Web</h3>
<ul>
<li>설치 필요 없음</li>
<li>단점: 기기기능(GPS/카메라 등) 제한</li>
</ul>
<h3>▲ Native App</h3>
<ul>
<li>빠르고 기능 사용 가능</li>
<li>Android/iOS 각각 개발 필요</li>
</ul>
<h3>▲ Hybrid App</h3>
<ul>
<li>웹 기반 + Native 기능 혼합</li>
<li>장점: 개발/수정 빠름</li>
<li>단점: 일부 기능 느림</li>
</ul>
