9:T1921,<h1>모바일 보안 기초 정리 (Android &#x26; iOS)</h1>
<h2>1. 운영체제 개념</h2>
<p>| 구분                | Android (갤럭시)        | iOS (아이폰)                   |
| ------------------- | ----------------------- | ------------------------------ |
| 기반 OS             | Linux 기반              | Unix 기반 (Darwin)             |
| 사용자 커스터마이징 | 높음                    | 낮음                           |
| 앱 설치 방식        | APK 파일 직접 설치 가능 | App Store 우선, 외부 설치 제한 |
| 루팅/탈옥           | 비교적 쉬움             | 매우 어려움                    |</p>
<hr>
<h1>2. 루팅(Rooting) &#x26; 탈옥(Jailbreak)</h1>
<h2>Android 루팅 이유</h2>
<ul>
<li>기본 UI가 마음에 들지 않음 → 커스텀 UI</li>
<li>오버클럭/언더클럭으로 성능 조절</li>
<li>배터리 최적화</li>
<li>통신사 기본 앱(블로트웨어) 삭제</li>
<li>시스템 파일 수정 가능</li>
</ul>
<h2>iOS 탈옥 이유</h2>
<ul>
<li>없는 기능 추가(예: 뒤로가기 제스처)</li>
<li>제한된 부분 해제</li>
<li>앱스토어 유료앱을 외부에서 설치 가능  (악성 가능성 매우 높음)</li>
</ul>
<h2>공통 위험</h2>
<ul>
<li>악성 APK / IPA 설치 → Spyware에 노출</li>
<li>금융앱·보안앱 실행 불가(루팅 탐지)</li>
<li>시스템 보안 강도 저하</li>
</ul>
<hr>
<h1>3. Spyware &#x26; SpyApp 공격 개념</h1>
<p>Spyware는 다음을 <strong>사용자 모르게</strong> 수집한다:</p>
<ul>
<li>GPS 위치</li>
<li>통화 기록</li>
<li>문자·카톡 기록</li>
<li>인터넷 검색 내역</li>
<li>사진 및 파일 접근</li>
</ul>
<p>→ 앱 목록에도 표시되지 않는 경우가 많음.</p>
<hr>
<h1>4. 제로 클릭 공격 (Zero-Click Attack)</h1>
<p>사용자가 직접 클릭하지 않아도 감염되는 공격.</p>
<p>예:</p>
<ul>
<li>받은 이미지 파일 보기만 해도 감염</li>
<li>문서 파일 열기</li>
<li>메시지 앱 취약점 이용 (MMS, iMessage 등)</li>
</ul>
<p>→ 또한 <strong>피싱 링크 클릭</strong>도 주요 감염 벡터.</p>
<hr>
<h1>5. 고객사 앱 보안 테스트 흐름</h1>
<ol>
<li>앱 분석</li>
<li>앱 변조(Modding)</li>
<li>앱과 서버 간 통신 분석</li>
<li>통신 패킷 변조</li>
<li>서버 공격·취약점 진단</li>
</ol>
<hr>
<h1>6. 모바일 개발 언어 &#x26; 리버싱 기본</h1>
<h2>Android</h2>
<ul>
<li>언어: <strong>Java, Kotlin</strong></li>
<li>컴파일: <code>.class</code> → DEX → Dalvik/ART VM</li>
<li>디컴파일: <strong>쉽다</strong> → Java 코드 거의 복원됨</li>
<li>Native 라이브러리: <code>.so</code> (C/C++ 빌드)</li>
<li>구조 분석 난이도: Android > iOS (초보 기준)</li>
</ul>
<h2>iOS</h2>
<ul>
<li>언어: <strong>Swift, Objective-C</strong></li>
<li>바이너리(Mach-O)로 컴파일 → 디컴파일 매우 어려움</li>
<li>Swift는 Mangle(Symbol 변환)로 함수 이름 난독화</li>
<li>앱 확장자: <strong>.ipa</strong></li>
<li>탈옥 기기 필요</li>
</ul>
<hr>
<h1>7. 모바일 리버싱 난이도 비교</h1>
<p>| 항목               | Android                  | iOS                            |
| ------------------ | ------------------------ | ------------------------------ |
| 언어               | Java/Kotlin              | Swift/Obj-C                    |
| 코드를 얻는 난이도 | 쉬움(디컴파일 잘됨)      | 어려움(바이너리 분석)          |
| 실행 파일          | APK                      | IPA                            |
| Native 코드        | JNI(.so)                 | 거의 모두 Native               |
| 에뮬레이터         | 가능 (Nox, Bluestack 등) | 어려움(맥 필요)                |
| 보안               | 중간                     | 매우 강함 (탈옥 자체가 어려움) |</p>
<hr>
<h1>8. Rooting / Jailbreak 도구</h1>
<h3>Android 루팅 도구</h3>
<ul>
<li><strong>Magisk</strong> (가장 대표적)</li>
<li>KernelSU</li>
<li>APatch</li>
<li>Custom Recovery (TWRP)</li>
<li>패치된 boot.img 사용</li>
</ul>
<h3>️ 주의: 벽돌(Brick) 가능</h3>
<ul>
<li>잘못하면 기기 부팅 불가</li>
<li>해결법: Recovery → Wipe → Firmware 재설치</li>
</ul>
<hr>
<h1>9. 실습에 필요한 개념</h1>
<h2>Recovery Mode</h2>
<ul>
<li>갤럭시: 전원 + 볼륨하 + 빅스비</li>
<li>기능:
<ul>
<li>Wipe data/factory reset</li>
<li>Wipe cache</li>
<li>Firmware 초기화</li>
</ul>
</li>
</ul>
<h2>앱 설치 구조</h2>
<ul>
<li>Android: <code>/sdcard/</code> 접근 가능</li>
<li>Root 권한 → <code>/</code> 전체 접근 가능</li>
<li>앱은 각자 sandbox에서 실행</li>
</ul>
<hr>
<h1>10. 앱의 ID 구조: 패키지명 (Package Name)</h1>
<p>예:</p>
<pre><code>com.facebook.app
</code></pre>
<p>규칙:</p>
<pre><code>도메인 반대로 → 프로젝트 → 앱이름
kr.co.company.appname
</code></pre>
<p>패키지명이 같으면?
→ 다른 앱이 기존 앱을 <strong>업데이트</strong>처럼 덮어쓰기 가능
→ 악성앱 공격 벡터 중 하나</p>

</code></pre>

</code></pre>
<p>언어 설정만 바꿔도 앱 내용이 다르게 표시됨.</p>
