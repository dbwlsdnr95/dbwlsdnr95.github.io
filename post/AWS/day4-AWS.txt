a:T2d76,<h1>AWS 네트워크 서비스 (VPC)</h1>
<hr>
<h2>1. Amazon VPC (Virtual Private Cloud)</h2>
<h3>1.1 VPC 개요</h3>
<ul>
<li>AWS 클라우드 내에서 <strong>논리적으로 격리된 가상 네트워크</strong></li>
<li>사용자가 직접 네트워크 구조를 설계 가능</li>
<li>IP 주소 범위, 서브넷, 라우팅, 보안 설정 제어</li>
</ul>
<h3>핵심 특징</h3>
<ul>
<li><strong>리전 단위</strong>로 생성</li>
<li>CIDR 블록으로 IP 범위 지정</li>
<li>완전한 네트워크 제어권 제공</li>
</ul>
<hr>
<h2>2. CIDR (Classless Inter-Domain Routing)</h2>
<h3>개요</h3>
<ul>
<li>IP 주소 범위를 표현하는 방식</li>
<li>형식: <code>IP주소/프리픽스</code></li>
</ul>
<h3>예시</h3>
<pre><code>10.0.0.0/16
</code></pre>
<ul>
<li>총 IP 개수: 65,536</li>
<li>VPC 생성 시 필수 설정</li>
</ul>
<hr>
<h2>3. 서브넷 (Subnet)</h2>
<h3>3.1 개요</h3>
<ul>
<li>VPC 내부를 나눈 <strong>IP 주소 범위</strong></li>
<li><strong>가용 영역(AZ) 단위</strong>로 생성됨</li>
</ul>
<h3>서브넷 유형</h3>
<ul>
<li><strong>퍼블릭 서브넷</strong>
<ul>
<li>인터넷 게이트웨이와 연결</li>
<li>EC2에 공인 IP 할당 가능</li>
</ul>
</li>
<li><strong>프라이빗 서브넷</strong>
<ul>
<li>외부 인터넷 직접 접근 불가</li>
<li>DB 서버 등에 사용</li>
</ul>
</li>
</ul>
<hr>
<h2>4. 인터넷 게이트웨이 (Internet Gateway)</h2>
<h3>개요</h3>
<ul>
<li>VPC와 인터넷 간 통신을 가능하게 하는 구성 요소</li>
</ul>
<h3>특징</h3>
<ul>
<li>퍼블릭 서브넷 필수 요소</li>
<li>공인 IP가 있는 리소스만 인터넷 접근 가능</li>
<li>VPC에 <strong>1개만 연결 가능</strong></li>
</ul>
<hr>
<h2>5. 라우팅 테이블 (Route Table)</h2>
<h3>개요</h3>
<ul>
<li>네트워크 트래픽의 <strong>경로를 정의</strong></li>
<li>각 서브넷은 반드시 하나의 라우팅 테이블과 연결</li>
</ul>
<h3>기본 라우트 예시</h3>
<pre><code>0.0.0.0/0 → Internet Gateway
</code></pre>
<hr>
<h2>6. NAT Gateway</h2>
<h3>개요</h3>
<ul>
<li><strong>프라이빗 서브넷 리소스가 외부 인터넷에 접근</strong>하도록 지원</li>
<li>인바운드 트래픽은 차단</li>
</ul>
<h3>특징</h3>
<ul>
<li>퍼블릭 서브넷에 위치</li>
<li>공인 IP(EIP) 필요</li>
<li>관리형 서비스 (고가용성)</li>
</ul>
<hr>
<h2>7. 보안 그룹 (Security Group)</h2>
<h3>개요</h3>
<ul>
<li><strong>가상 방화벽</strong></li>
<li>인스턴스 수준에서 동작</li>
</ul>
<h3>특징</h3>
<ul>
<li><strong>상태 저장(Stateful)</strong></li>
<li>허용 규칙만 설정 (거부 규칙 없음)</li>
<li>인바운드/아웃바운드 규칙 정의</li>
</ul>
<hr>
<h2>8. 네트워크 ACL (NACL)</h2>
<h3>개요</h3>
<ul>
<li>서브넷 수준에서 적용되는 보안 정책</li>
</ul>
<h3>특징</h3>
<ul>
<li><strong>상태 비저장(Stateless)</strong></li>
<li>허용 + 거부 규칙 모두 설정 가능</li>
<li>규칙 번호 순서대로 평가</li>
</ul>
<hr>
<h2>9. 보안 그룹 vs NACL</h2>
<p>| 구분      | 보안 그룹 | NACL        |
| --------- | --------- | ----------- |
| 적용 대상 | 인스턴스  | 서브넷      |
| 상태      | Stateful  | Stateless   |
| 규칙      | 허용만    | 허용 + 거부 |
| 평가 방식 | 전체      | 순서        |</p>
<hr>
<h2>10. VPC Peering</h2>
<h3>개요</h3>
<ul>
<li>두 VPC 간 <strong>프라이빗 통신</strong> 연결</li>
</ul>
<h3>특징</h3>
<ul>
<li>다른 리전 / 다른 계정 가능</li>
<li><strong>전이적 연결 불가</strong></li>
<li>CIDR 겹치면 불가</li>
</ul>
<hr>
<h2>11. AWS VPN</h2>
<h3>개요</h3>
<ul>
<li>온프레미스 ↔ AWS VPC 간 <strong>암호화된 연결</strong></li>
</ul>
<h3>유형</h3>
<ul>
<li><strong>Site-to-Site VPN</strong></li>
<li><strong>Client VPN</strong></li>
</ul>
<hr>
<h2>12. AWS Direct Connect</h2>
<h3>개요</h3>
<ul>
<li>온프레미스와 AWS 간 <strong>전용 회선 연결</strong></li>
</ul>
<h3>특징</h3>
<ul>
<li>안정적</li>
<li>낮은 지연 시간</li>
<li>대규모 트래픽에 적합</li>
</ul>
<hr>
<h2>13. VPC 엔드포인트 (VPC Endpoint)</h2>
<h3>개요</h3>
<ul>
<li>인터넷을 거치지 않고 AWS 서비스에 연결</li>
</ul>
<h3>유형</h3>
<ul>
<li><strong>Gateway Endpoint</strong>
<ul>
<li>S3, DynamoDB</li>
</ul>
</li>
<li><strong>Interface Endpoint</strong>
<ul>
<li>PrivateLink 기반</li>
<li>대부분 AWS 서비스</li>
</ul>
</li>
</ul>
<hr>
<h2>14. Elastic Load Balancing (ELB)</h2>
<h3>개요</h3>
<ul>
<li>트래픽을 여러 대상에 분산</li>
</ul>
<h3>유형</h3>
<ul>
<li><strong>ALB (Application Load Balancer)</strong>
<ul>
<li>HTTP/HTTPS</li>
<li>Layer 7</li>
</ul>
</li>
<li><strong>NLB (Network Load Balancer)</strong>
<ul>
<li>TCP/UDP</li>
<li>고성능</li>
</ul>
</li>
<li><strong>CLB</strong>
<ul>
<li>레거시</li>
</ul>
</li>
</ul>
<hr>
<h2>15. Route 53 (DNS 서비스)</h2>
<h3>개요</h3>
<ul>
<li>AWS의 <strong>관리형 DNS 서비스</strong></li>
</ul>
<h3>주요 기능</h3>
<ul>
<li>도메인 등록</li>
<li>트래픽 라우팅</li>
<li>헬스 체크</li>
</ul>
<hr>
<h2>16. 네트워크 아키텍처 핵심 정리 (시험 포인트)</h2>
<ul>
<li>VPC = 가상 네트워크</li>
<li>Subnet = AZ 단위</li>
<li>Public Subnet = IGW 연결</li>
<li>Private Subnet = NAT Gateway</li>
<li>Security Group = 인스턴스 방화벽</li>
<li>NACL = 서브넷 방화벽</li>
<li>Direct Connect = 전용 회선</li>
<li>VPC Endpoint = 인터넷 없이 AWS 서비스 접근</li>
</ul>
<hr>
<h2>17. Amazon CloudFront</h2>
<h3>17.1 개요</h3>
<ul>
<li>AWS의 <strong>CDN(Content Delivery Network)</strong> 서비스</li>
<li>전 세계에 분산된 <strong>엣지 로케이션(Edge Location)</strong> 을 통해 콘텐츠 제공</li>
<li>사용자와 가장 가까운 위치에서 콘텐츠 전달 → <strong>지연 시간 감소</strong></li>
</ul>
<hr>
<h3>17.2 제공 콘텐츠 유형</h3>
<ul>
<li><strong>정적 콘텐츠</strong>
<ul>
<li>이미지, CSS, JS, HTML</li>
</ul>
</li>
<li><strong>동적 콘텐츠</strong>
<ul>
<li>API 응답</li>
<li>로그인 페이지</li>
<li>실시간 데이터</li>
</ul>
</li>
</ul>
<hr>
<h3>17.3 주요 구성 요소</h3>
<ul>
<li><strong>Origin</strong>
<ul>
<li>원본 서버</li>
<li>S3, EC2, ALB, 외부 서버 가능</li>
</ul>
</li>
<li><strong>Distribution</strong>
<ul>
<li>CloudFront 설정 단위</li>
<li>Web Distribution 사용</li>
</ul>
</li>
<li><strong>Edge Location</strong>
<ul>
<li>캐시 저장 위치</li>
<li>사용자 요청을 최초로 처리</li>
</ul>
</li>
</ul>
<hr>
<h3>17.4 보안 기능</h3>
<ul>
<li>HTTPS 기본 지원</li>
<li><strong>AWS Shield Standard</strong> 기본 제공</li>
<li><strong>AWS WAF 연동 가능</strong></li>
<li><strong>서명된 URL / 서명된 쿠키</strong>
<ul>
<li>콘텐츠 접근 제어</li>
</ul>
</li>
</ul>
<hr>
<h3>17.5 사용 사례</h3>
<ul>
<li>글로벌 정적 웹 사이트</li>
<li>미디어 스트리밍</li>
<li>API 가속</li>
<li>트래픽 폭증 대응</li>
</ul>
<hr>
<h2>18. AWS Global Accelerator</h2>
<h3>18.1 개요</h3>
<ul>
<li>AWS 글로벌 네트워크 기반 <strong>애플리케이션 가속 서비스</strong></li>
<li>사용자 → 가장 가까운 엣지 → AWS 백본 네트워크</li>
</ul>
<hr>
<h3>18.2 주요 특징</h3>
<ul>
<li><strong>고정 Anycast IP 주소 제공</strong></li>
<li>TCP / UDP 트래픽 가속</li>
<li>상태 기반 헬스 체크</li>
<li>자동 장애 조치</li>
</ul>
<hr>
<h3>18.3 CloudFront vs Global Accelerator</h3>
<p>| 항목      | CloudFront  | Global Accelerator       |
| --------- | ----------- | ------------------------ |
| 목적      | 콘텐츠 전달 | 네트워크 가속            |
| 캐싱      | O           | X                        |
| 프로토콜  | HTTP/HTTPS  | TCP/UDP                  |
| IP 주소   | 변경 가능   | 고정 Anycast             |
| 사용 대상 | 웹 콘텐츠   | 게임, API, 실시간 서비스 |</p>
<hr>

<h2>23. AWS Well-Architected Framework</h2>
