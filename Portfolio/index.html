Project Description

Spring4Shell (CVE-2022-22965)은 Spring Framework Core에서 발생한 심각한 원격 코드 실행(RCE) 취약점입니다. 본 프로젝트는 해당 취약점의 동작 원리를 심층 분석하고, 실제 공격 시나리오(WebShell Upload)를 재현하여 방어 전략을 수립하기 위해 진행되었습니다.

Project Purpose

  • Spring Core의 DataBinder 메커니즘과 취약점 발생 원리 이해
  • JDK 9+ 환경에서의 우회 공격 기법 분석
  • 실제 공격 재현을 통한 파급력 검증 및 보안 대응책 마련

Assigned Role

  • Vulnerability Analysis: CVE-2022-22965의 원인이 되는 `CachedIntrospectionResults` 코드 분석
  • PoC Development: Tomcat AccessLogValve를 악용한 WebShell 생성 공격 코드 작성
  • Verification: Burp Suite를 활용한 공격 시나리오 검증 및 WAF 우회 가능성 테스트

Key Achievements

  • Spring Boot 2.6.5 + Java 11 환경에서 RCE 공격 성공 및 재현
  • WebShell 업로드를 통한 시스템 제어권 탈취 시나리오 입증
  • 다양한 WAF 벤더(AWS, Akamai)의 탐지 패턴 식별 및 우회 기법 연구
