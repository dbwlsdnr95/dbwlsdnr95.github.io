# 💡 AWS EC2 ~ 스토리지 구간 핵심 정리 

------

# #1 EC2 (Amazon Elastic Compute Cloud)

## ✅ 개념

- AWS에서 제공하는 **가상 서버(Compute 인스턴스)** 서비스
- 필요할 때 몇 분 만에 서버 생성 가능
- 사용한 만큼만 비용 지불 (온디맨드)

------

## 🖥️ EC2 인스턴스 구성 요소

| 구성요소       | 설명                              |
| -------------- | --------------------------------- |
| AMI            | OS + 소프트웨어 패키지 이미지     |
| Instance Type  | CPU, RAM, 네트워크 성능 크기      |
| Storage        | EBS / Instance Store              |
| Key Pair       | SSH 접속용 인증키                 |
| Security Group | 인바운드/아웃바운드 방화벽        |
| IAM Role       | 인스턴스가 AWS 자원에 접근할 권한 |

------

## 🔥 EC2 과금 요소

- 인스턴스 타입(가장 중요)
- OS 종류
- 온디맨드 / 리저브드 / 스팟
- EBS 스토리지 비용
- 데이터 전송량(인터넷 아웃바운드)

------

## ⚙️ EC2 구매 옵션

### 1) **온디맨드(On-Demand)**

- 사용한 시간만큼 지불
- 단기, 테스트 환경에 적합
- 가장 비쌈

### 2) **리저브드 인스턴스(RI)**

- 1년/3년 약정 → 최대 75% 절감
- 안정적인 워크로드에 적합

### 3) **스팟 인스턴스(Spot)**

- AWS 남는 자원을 경매 방식 제공
- 최대 90% 할인
- 갑자기 중지될 수 있어 배치 작업/비핵심 작업용

------

## 🔐 Security Group

- EC2 인스턴스 수준 방화벽
- 인바운드/아웃바운드 규칙 설정
- **상태 저장(Stateful)**
   → 인바운드 허용하면 응답 아웃바운드는 자동 허용

------

## 🛠️ EC2 핵심 기능

- Auto Scaling → 서버 자동 증가/감소
- Elastic Load Balancer(ELB) → 트래픽 분산
- Placement Group → 고성능 컴퓨팅(HPC)

------

------

# #2 EC2 스토리지 개요

EC2에서 사용할 수 있는 스토리지는 크게 **3종류**:

1. **EBS(Elastic Block Store)**
2. **Instance Store**
3. **EFS(Elastic File System)**

그리고 별도로 **S3**도 많이 사용되므로 함께 정리.

------

# #3 EBS (Elastic Block Store)

## 📌 특징

- EC2 인스턴스에 연결하는 **네트워크 블록 스토리지**
- 인스턴스 종료해도 데이터 유지됨
- 스냅샷을 통한 복구 및 백업 가능

------

## 📦 EBS 타입

| 타입        | 특징                       | 사용 용도                 |
| ----------- | -------------------------- | ------------------------- |
| **gp3**     | 범용 SSD, 표준 성능        | 대부분의 워크로드         |
| **io1/io2** | 고성능 SSD, IOPS 지정 가능 | DB, 트랜잭션 높은 시스템  |
| **st1**     | HDD, 높은 처리량           | 빅데이터 분석             |
| **sc1**     | 저비용 HDD                 | 자주 접근하지 않는 데이터 |

------

## 🔄 EBS 스냅샷

- S3에 저장되는 백업
- 증분 백업 방식
- AMI 생성의 기반

------

# #4 Instance Store

## ⚠️ 특징

- EC2 물리 서버에 연결된 **임시 스토리지**
- 인스턴스 **중지/종료 시 데이터 사라짐**
- 고성능 I/O 가능

### 언제 사용?

- 캐시
- 임시 데이터 처리
- 성능이 매우 중요한 경우(HPC)

------

# #5 EFS (Elastic File System)

## 📌 특징

- Linux 전용 **NFS 기반 네트워크 파일 시스템**
- 여러 EC2 인스턴스가 동시에 마운트 가능
- 자동 확장/축소
- 고비용(비싸지만 확장성 뛰어남)

### 사용 예시

- 웹서버 여러 대가 동일한 `/var/www/html`을 공유
- 애플리케이션 공유 파일 스토리지

------

# #6 S3 (Simple Storage Service)

## 📌 특징

- 무한 확장 가능한 **오브젝트 스토리지**
- 11 9’s (99.999999999%) 내구성
- 서버리스
- 전 세계 어디서나 접근 가능

------

## 🗂️ 버킷(Bucket)

- S3 최상위 컨테이너
- 이름이 전 세계에서 유일해야 함

------

## 🔐 S3 보안 방식

1. **IAM 정책**
2. **버킷 정책**
3. **ACL(거의 사용하지 않음)**
4. **암호화(SSE-S3, SSE-KMS)**

------

## 🗃️ S3 스토리지 클래스

| 클래스                     | 설명                 |
| -------------------------- | -------------------- |
| **Standard**               | 자주 접근하는 데이터 |
| **IA (Infrequent Access)** | 가끔 조회            |
| **Glacier Instant**        | 저렴한 아카이브      |
| **Glacier / Deep Glacier** | 장기 보관            |
| **One Zone IA**            | 단일 AZ 저장(저렴)   |

------

## 🚀 S3 버전 관리

- Enable Versioning → 데이터 손실 방지
- 삭제 방지도 가능(Delete Marker)

------

# #7 EC2 vs EBS vs EFS vs S3 비교

| 항목    | EC2                  | EBS                    | EFS              | S3               |
| ------- | -------------------- | ---------------------- | ---------------- | ---------------- |
| 유형    | Compute              | Block Storage          | File Storage     | Object Storage   |
| 지속성  | 인스턴스 종료와 무관 | 유지됨                 | 유지됨           | 무한             |
| 사용 예 | 서버 실행            | DB 저장, 시스템 디스크 | 공유 파일 시스템 | 이미지/로그/백업 |

------

# #8 Cloud Practitioner 시험 포인트 정리

### ⭐ 가장 중요하게 출제되는 개념

- EBS는 **AZ 내에서만** 작동 (Cross-AZ 불가)
- EFS는 여러 EC2에서 **동시에 마운트 가능**
- S3는 오브젝트 스토리지 (파일 시스템 아님)
- EC2 종료해도 EBS는 유지됨
- Instance Store는 종료하면 **데이터 삭제**

------

# ✔️ 완전 요약(한 방에 외우는 버전)

```
EC2 = 가상 서버  
EBS = EC2 전용 블록 스토리지  
Instance Store = 임시 로컬 스토리지  
EFS = 여러 EC2가 공유하는 파일 스토리지  
S3 = 무한 확장, 오브젝트 스토리지
```
