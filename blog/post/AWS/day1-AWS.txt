9:T1829,<h1>AWS Basic → IAM 전체 정리</h1>
<h1>#1. 클라우드 컴퓨팅 기본 개념</h1>
<h2>클라우드 컴퓨팅이란?</h2>
<blockquote>
<p><strong>필요한 IT 자원을 인터넷을 통해 필요할 때 즉시 제공받는 서비스</strong></p>
</blockquote>
<p>특징:</p>
<ul>
<li>온디맨드(On-Demand)</li>
<li>탄력성(Elasticity)</li>
<li>확장성(Scalability)</li>
<li>사용한 만큼만 비용 지불(Pay-as-you-go)</li>
<li>관리 부담 감소 (AWS가 인프라 운영)</li>
</ul>
<hr>
<h1>#2. 클라우드 배포 모델</h1>
<h3>Public Cloud</h3>
<ul>
<li>AWS, Azure, GCP 등</li>
<li>다수 고객이 공유 인프라를 사용</li>
<li>저렴하고 확장 쉬움</li>
</ul>
<h3>Private Cloud</h3>
<ul>
<li>기업 내부 전용 클라우드</li>
<li>높은 보안, 비용은 큼</li>
</ul>
<h3>Hybrid Cloud</h3>
<ul>
<li>Public + Private 조합</li>
<li>민감한 데이터는 Private, 나머지는 Public</li>
</ul>
<hr>
<h1>#3. AWS 글로벌 인프라</h1>
<h2>리전(Region)</h2>
<ul>
<li>AWS 인프라가 있는 물리적 위치 (한국: <strong>ap-northeast-2</strong>)</li>
<li>재해 대비/법적 규제/지연 시간 고려하여 선택</li>
</ul>
<h2>가용 영역(AZ, Availability Zone)</h2>
<ul>
<li>같은 리전 내의 독립적인 데이터센터</li>
<li>장애 대비 위해 최소 2개 AZ 사용 권장</li>
</ul>
<h2>Edge Location</h2>
<ul>
<li>CloudFront CDN이 사용하는 전 세계 캐시 서버</li>
</ul>
<hr>
<h1>#4. AWS 공유 책임 모델 (Shared Responsibility Model)</h1>
<h2>AWS 책임 (Security <strong>of</strong> the Cloud)</h2>
<ul>
<li>하드웨어, 네트워크, 물리적 보안</li>
<li>가상화 계층 관리</li>
</ul>
<h2>고객 책임 (Security <strong>in</strong> the Cloud)</h2>
<ul>
<li>IAM 권한 관리</li>
<li>OS, 애플리케이션 보안 패치</li>
<li>데이터 보호, 암호화 설정</li>
</ul>
<blockquote>
<p>즉, AWS는 인프라를, 고객은 설정을 책임진다.</p>
</blockquote>
<hr>
<h1>#5. AWS 핵심 서비스(간단 정리)</h1>
<p>| 영역       | 서비스              | 설명                  |
| ---------- | ------------------- | --------------------- |
| Compute    | <strong>EC2</strong>             | 가상 서버             |
| Storage    | <strong>S3</strong>              | 객체 스토리지         |
| Database   | <strong>RDS / DynamoDB</strong>  | 관계형 / NoSQL DB     |
| Networking | <strong>VPC</strong>             | 네트워크 구성         |
| Security   | <strong>IAM / KMS / WAF</strong> | 권한, 암호화, 웹 보안 |</p>
<hr>
<h1>#6. IAM (Identity and Access Management)</h1>
<p>IAM은 AWS 리소스에 대한 접근을 제어하는 <strong>보안 핵심 서비스</strong>이다.</p>
<hr>
<h1>## IAM 구성 요소</h1>
<h2>1. 사용자(User)</h2>
<ul>
<li>사람 또는 시스템용 계정</li>
<li>비밀번호(콘솔), Access Key(API/CLI) 사용 가능</li>
</ul>
<hr>
<h2>2. 그룹(Group)</h2>
<ul>
<li>여러 사용자에게 동일 권한을 부여하기 위한 묶음</li>
<li>그룹 자체로 로그인은 불가</li>
</ul>
<hr>
<h2>3. 정책(Policy)</h2>
<p>AWS에서 가장 중요한 부분.</p>
<ul>
<li>JSON 형태의 <strong>권한 문서</strong></li>
<li>“누가 무엇을 할 수 있는지” 정의</li>
</ul>
<h3>예시 (S3 ReadOnly 정책)</h3>
<pre><code>{
  "Effect": "Allow",
  "Action": ["s3:Get*", "s3:List*"],
  "Resource": "*"
}
</code></pre>
<hr>
<h2>4. 역할(Role)</h2>
<ul>
<li>사람이 사용하지 않음</li>
<li>AWS 리소스가 다른 AWS 서비스에 접근할 수 있도록 하는 자격증명</li>
</ul>
<p>예:</p>
<ul>
<li>EC2 → S3 접근을 허용하는 IAM Role</li>
<li>Lambda → DynamoDB 접근 허용 Role</li>
</ul>
<hr>
<h1>## IAM 보안 베스트 프랙티스</h1>
<h3>Root 계정 사용 금지</h3>
<ul>
<li>MFA 활성화 후 보관</li>
<li>결제, 계정 설정 외에는 사용하지 않음</li>
</ul>
<h3>최소 권한 원칙(Least Privilege)</h3>
<ul>
<li>필요한 리소스만 최소 권한으로 허용</li>
</ul>
<h3>MFA 활성화</h3>
<ul>
<li>사용자별 MFA 적용 필수</li>
</ul>
<h3>Access Key 노출 금지</h3>
<ul>
<li>사용하지 않는 키는 즉시 삭제</li>
<li>역할(Role) 우선 사용</li>
</ul>
<h3>CloudTrail 활성화</h3>
<ul>
<li>IAM 변경 사항 및 API 호출 기록 모니터링</li>
</ul>
<hr>
<h1>## IAM 정책 유형</h1>
<p>| 정책 타입        | 설명                              |
| ---------------- | --------------------------------- |
| AWS 관리형 정책  | AWS가 제공하는 표준 정책          |
| 고객 관리형 정책 | 직접 정의한 JSON 정책             |
| 인라인 정책      | 특정 사용자/역할/그룹에 직접 삽입 |</p>
<hr>
<h1>## IAM 인증 방식</h1>
<p>| 방식            | 설명                        |
| --------------- | --------------------------- |
| 비밀번호        | 콘솔 로그인                 |
| Access Key      | CLI/API 요청                |
| Temporary Token | STS를 통한 일시적 권한 부여 |</p>
<hr>
<h1>## IAM 핵심 서비스 연결 흐름</h1>
<pre><code>User → IAM User → Policy → API/Console → AWS Resource
</code></pre>
<p>역할(Role) 사용 시:</p>
<pre><code>EC2 → IAM Role → Policy → AWS Resource
</code></pre>
<hr>
<h1>#7. IAM 관련 AWS 시험 포인트</h1>
<ul>
<li>IAM은 리전(global) 서비스이다 (전 세계 동일하게 적용)</li>
<li>최소 권한 모델 기반 운영(LP: Least Privilege)</li>
<li>Root 계정은 가능한 한 사용하지 않는다</li>
<li>사용자 그룹으로 권한을 관리하는 것이 효율적</li>
<li>EC2 인스턴스에 Access Key 넣지 말 것 → 반드시 <strong>IAM Role</strong> 사용</li>
</ul>
<p>시험에 아주 자주 나오는 포인트들이다.</p>
<hr>
<h1>마무리 요약</h1>
<p>IAM은 AWS 보안의 기초이며,
다음과 같은 질문에 답할 수 있어야 한다:</p>
<ul>
<li>누가 로그인할 수 있는가? (사용자)</li>
<li>어떤 권한을 주는가? (정책)</li>
<li>어떤 방식으로 AWS 서비스에 접근하는가? (자격증명)</li>
<li>어떤 서비스가 대신 다른 서비스에 접근하는가? (역할)</li>
</ul>
