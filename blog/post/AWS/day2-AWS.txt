9:T430b,<h1>Elastic Load Balancing &#x26; Auto Scaling</h1>
<h2>시스템 아키텍처 핵심 개념</h2>
<h3>확장성 (Scalability)</h3>
<ul>
<li><strong>수직 확장 (Vertical Scaling)</strong>
<ul>
<li>인스턴스 사양 자체를 상향</li>
<li>예: t2.micro → t2.large</li>
<li>시스템 중단 필요</li>
</ul>
</li>
<li><strong>수평 확장 (Horizontal Scaling)</strong>
<ul>
<li>인스턴스 개수 자체를 증가</li>
<li>시스템 중단 없이 확장 가능</li>
<li>클라우드 환경에서 일반적으로 사용</li>
</ul>
</li>
</ul>
<hr>
<h3>고가용성 (High Availability)</h3>
<ul>
<li>장애 상황에서도 지속적으로 정상 서비스 제공</li>
<li>여러 <strong>가용 영역(AZ)</strong> 에 시스템 분산 배치</li>
<li>하나의 AZ 장애 시 다른 AZ에서 서비스 유지</li>
</ul>
<hr>
<h3>느슨한 결합 (Loosely Coupled Architecture)</h3>
<ul>
<li>한 구성 요소 장애가 다른 요소에 최소한만 영향</li>
<li>로드 밸런서를 통해 트래픽 분산</li>
<li>확장성과 장애 대응에 유리</li>
</ul>
<hr>
<h2>Elastic Load Balancing (ELB)</h2>
<h3>로드 밸런서 개요</h3>
<ul>
<li>네트워크 트래픽을 여러 대상에 자동 분산</li>
<li>비정상 대상은 자동으로 트래픽 제외</li>
<li>대상이 정상화되면 자동 복구</li>
</ul>
<hr>
<h3>ELB 종류</h3>
<h4>Application Load Balancer (ALB)</h4>
<ul>
<li>L7 (HTTP/HTTPS)</li>
<li>URL, 헤더 기반 라우팅</li>
<li>웹 애플리케이션에 적합</li>
</ul>
<h4>Network Load Balancer (NLB)</h4>
<ul>
<li>L4 (TCP/UDP/TLS)</li>
<li>대규모 트래픽 처리</li>
<li>초저지연 네트워크</li>
</ul>
<h4>Classic Load Balancer</h4>
<ul>
<li>이전 세대 로드 밸런서</li>
<li>EC2-Classic 환경 전용</li>
</ul>
<h4>Gateway Load Balancer (GWLB)</h4>
<ul>
<li>L3/L4</li>
<li>방화벽, IDS/IPS 같은 보안 어플라이언스 연동</li>
<li>GENEVE 프로토콜 사용</li>
</ul>
<hr>
<h3>Target Group 구성</h3>
<ul>
<li>트래픽을 전달할 대상의 집합</li>
<li>대상 유형
<ul>
<li>EC2 인스턴스</li>
<li>Auto Scaling Group</li>
<li>IP 주소</li>
<li>Lambda 함수</li>
</ul>
</li>
</ul>
<h3>Health Check</h3>
<ul>
<li>대상의 정상 여부 주기적 확인</li>
<li>비정상 대상은 자동 제외</li>
</ul>
<hr>
<h2>EC2 Auto Scaling</h2>
<h3>개요</h3>
<ul>
<li>트래픽 수요에 따라 EC2 인스턴스를 자동으로 확장/축소</li>
<li>Scale Out / Scale In 자동 수행</li>
</ul>
<hr>
<h3>Auto Scaling 구성 요소</h3>
<ul>
<li><strong>Auto Scaling Group</strong></li>
<li><strong>Launch Template</strong></li>
<li><strong>조정 정책 (Scaling Policy)</strong></li>
</ul>
<hr>
<h3>조정 방식</h3>
<h4>고정 용량 유지</h4>
<ul>
<li>항상 동일한 인스턴스 수 유지</li>
</ul>
<h4>수동 조정</h4>
<ul>
<li>최소/최대/원하는 용량 직접 설정</li>
</ul>
<h4>일정 기반 조정</h4>
<ul>
<li>특정 시간·요일 기준 확장</li>
</ul>
<h4>동적 조정 (Dynamic Scaling)</h4>
<ul>
<li>CPU, 네트워크 등 지표 기반</li>
</ul>
<h5>Target Tracking</h5>
<ul>
<li>목표 지표 유지 (예: CPU 50%)</li>
</ul>
<h5>Step Scaling</h5>
<ul>
<li>단계별 증가/감소</li>
</ul>
<h5>Simple Scaling</h5>
<ul>
<li>단순 경보 기반 조정</li>
</ul>
<h4>예측 조정 (Predictive Scaling)</h4>
<ul>
<li>머신러닝 기반 용량 예측</li>
</ul>
<hr>
<h3>Auto Scaling 장점</h3>
<ul>
<li>비용 최적화</li>
<li>장애 인스턴스 자동 교체</li>
<li>Multi-AZ 기반 고가용성</li>
</ul>
<hr>
<h1>Serverless 컴퓨팅</h1>
<h2>Serverless 개념</h2>
<ul>
<li>서버 운영/관리 불필요</li>
<li>인프라 운영은 AWS가 담당</li>
<li>사용자는 코드와 비즈니스 로직에만 집중</li>
</ul>
<h3>대표 서비스</h3>
<ul>
<li>AWS Lambda</li>
<li>AWS Fargate</li>
<li>Amazon S3</li>
<li>DynamoDB</li>
<li>Aurora Serverless</li>
<li>SNS / SQS / API Gateway</li>
</ul>
<hr>
<h2>AWS Lambda</h2>
<h3>Lambda 개요</h3>
<ul>
<li>이벤트 기반 서버리스 컴퓨팅</li>
<li>요청이 있을 때만 실행</li>
<li>실행 횟수 + 실행 시간 기반 과금</li>
</ul>
<hr>
<h3>Lambda 특징</h3>
<ul>
<li>서버 관리 불필요</li>
<li>자동 확장</li>
<li>다양한 언어 지원
<ul>
<li>Python, Node.js, Java, C#, Ruby 등</li>
</ul>
</li>
</ul>
<hr>
<h3>Lambda 연계 구조</h3>
<ul>
<li>단독 사용보다 서비스 결합이 일반적</li>
<li>주요 연계 서비스
<ul>
<li>S3</li>
<li>API Gateway</li>
<li>DynamoDB</li>
<li>SNS / SQS</li>
<li>CloudFront</li>
</ul>
</li>
</ul>
<hr>
<h1>배치 &#x26; 컨테이너 컴퓨팅</h1>
<h2>AWS Batch</h2>
<h3>개요</h3>
<ul>
<li>대규모 배치 작업 실행 서비스</li>
<li>작업량에 따라 리소스 자동 프로비저닝</li>
<li>Docker 컨테이너 기반 실행</li>
</ul>
<h3>활용 사례</h3>
<ul>
<li>생명과학 분석</li>
<li>영상 렌더링</li>
<li>대규모 데이터 처리</li>
</ul>
<hr>
<h2>컨테이너 개념</h2>
<h3>Container</h3>
<ul>
<li>애플리케이션과 라이브러리를 패키징</li>
<li>OS 환경과 무관하게 실행 가능</li>
<li>독립적인 실행 환경</li>
<li>마이크로서비스 아키텍처에 적합</li>
</ul>
<hr>
<h2>AWS 컨테이너 서비스</h2>
<h3>Amazon ECS (Elastic Container Service)</h3>
<ul>
<li>AWS 자체 컨테이너 오케스트레이션</li>
<li>Docker 기반</li>
<li>간단한 운영</li>
</ul>
<hr>
<h3>Amazon EKS (Elastic Kubernetes Service)</h3>
<ul>
<li>AWS 관리형 Kubernetes</li>
<li>대규모 컨테이너 워크로드에 적합</li>
<li>오픈소스 Kubernetes 사용</li>
</ul>
<hr>
<h3>AWS Fargate</h3>
<ul>
<li>서버리스 컨테이너 실행 환경</li>
<li>서버/클러스터 관리 불필요</li>
<li>ECS 및 EKS와 함께 사용</li>
</ul>
<hr>
<h2>컨테이너 서비스 비교 요약</h2>
<p>| 구분           | ECS      | EKS        | Fargate   |
| -------------- | -------- | ---------- | --------- |
| 오케스트레이션 | AWS 자체 | Kubernetes | 서버리스  |
| 서버 관리      | 필요     | 필요       | 불필요    |
| 난이도         | 낮음     | 높음       | 매우 낮음 |
| 확장성         | 높음     | 매우 높음  | 자동      |</p>
<hr>
<h2>AWS Elastic Beanstalk</h2>
<h3>개요</h3>
<ul>
<li><strong>애플리케이션 배포를 자동화</strong>하는 PaaS 서비스</li>
<li>인프라 설정 없이 <strong>코드만 업로드</strong>하면 실행 환경 자동 구성</li>
</ul>
<hr>
<h3>Elastic Beanstalk 특징</h3>
<ul>
<li>EC2, Auto Scaling, ELB, RDS 등을 <strong>자동으로 프로비저닝</strong></li>
<li>사용자는 애플리케이션 코드에만 집중</li>
<li>인프라는 AWS가 관리하지만 <strong>EC2 자체 접근은 가능</strong></li>
</ul>
<hr>
<h3>지원 언어 / 플랫폼</h3>
<ul>
<li>Java</li>
<li>.NET</li>
<li>Python</li>
<li>Node.js</li>
<li>PHP</li>
<li>Ruby</li>
<li>Go</li>
<li>Docker</li>
</ul>
<hr>
<h3>Elastic Beanstalk 요약</h3>
<ul>
<li><strong>PaaS</strong></li>
<li>서버 접근 가능</li>
<li>빠른 애플리케이션 배포</li>
<li>운영 부담 감소</li>
</ul>
<hr>
<h3>시험 포인트</h3>
<ul>
<li>서버리스 아님</li>
<li>EC2 기반</li>
<li>인프라 자동 관리</li>
</ul>
<hr>
<h2>Amazon Lightsail</h2>
<h3>개요</h3>
<ul>
<li><strong>간단한 가상 서버(VPS)</strong> 서비스</li>
<li>AWS를 처음 사용하는 사용자 또는 소규모 서비스용</li>
</ul>
<hr>
<h3>Lightsail 특징</h3>
<ul>
<li>EC2보다 단순한 UI</li>
<li>월 정액 요금</li>
<li>서버, 스토리지, 네트워크 패키지 제공</li>
<li>WordPress, LAMP, Node.js 등 사전 구성 이미지 제공</li>
</ul>
<hr>
<h3>사용 사례</h3>
<ul>
<li>개인 프로젝트</li>
<li>소규모 웹사이트</li>
<li>테스트 서버</li>
<li>스타트업 초기 서비스</li>
</ul>
<hr>
<h3>Lightsail 요약</h3>
<ul>
<li>EC2의 간소화 버전</li>
<li>설정이 매우 쉬움</li>
<li>대규모 확장에는 부적합</li>
</ul>
<hr>
<h3>시험 포인트</h3>
<ul>
<li>초보자용 AWS 서비스</li>
<li>복잡한 인프라 설계용 아님</li>
</ul>
<hr>
<h2>Amazon WorkSpaces</h2>
<h3>개요</h3>
<ul>
<li><strong>클라우드 기반 가상 데스크톱(VDI)</strong> 서비스</li>
<li>어디서든 동일한 데스크톱 환경 제공</li>
</ul>
<hr>
<h3>WorkSpaces 특징</h3>
<ul>
<li>Windows / Linux 데스크톱 제공</li>
<li>기업용 원격 근무 환경</li>
<li>로컬 PC 데이터 저장 불필요</li>
<li>보안 강화 (데이터 중앙 집중 관리)</li>
</ul>
<hr>
<h3>사용 사례</h3>
<ul>
<li>재택근무</li>
<li>외부 협력사 환경 제공</li>
<li>보안이 중요한 업무 환경</li>
</ul>
<hr>
<h2>Amazon AppStream 2.0</h2>
<h3>개요</h3>
<ul>
<li><strong>애플리케이션 스트리밍 서비스</strong></li>
<li>데스크톱 전체가 아닌 <strong>앱 단위 제공</strong></li>
</ul>
<hr>
<h3>AppStream 2.0 특징</h3>
<ul>
<li>웹 브라우저로 애플리케이션 실행</li>
<li>클라이언트 설치 불필요</li>
<li>고성능 그래픽 앱 지원</li>
</ul>
<hr>
<h3>WorkSpaces vs AppStream 2.0</h3>
<p>| 항목      | WorkSpaces    | AppStream 2.0 |
| --------- | ------------- | ------------- |
| 제공 범위 | 전체 데스크톱 | 애플리케이션  |
| 사용 방식 | 지속 사용     | 필요 시       |
| 설치      | 필요 없음     | 필요 없음     |
| 목적      | 업무 환경     | 앱 제공       |</p>
<hr>
<h3>시험 포인트</h3>
<ul>
<li>데스크톱 → WorkSpaces</li>
<li>앱만 제공 → AppStream 2.0</li>
</ul>
<hr>
<h2>AWS Outposts</h2>
<h3>개요</h3>
<ul>
<li><strong>AWS 인프라를 온프레미스에 설치</strong></li>
<li>하이브리드 클라우드 구성</li>
</ul>
<hr>
<h3>Outposts 특징</h3>
<ul>
<li>AWS 서버/스토리지를 고객 데이터센터에 배치</li>
<li>AWS 콘솔로 동일하게 관리</li>
<li>낮은 지연 시간</li>
<li>데이터 로컬 처리 요구 사항 충족</li>
</ul>
<hr>
<h3>사용 사례</h3>
<ul>
<li>금융</li>
<li>공공기관</li>
<li>제조업</li>
<li>데이터 레지던시 요구 환경</li>
</ul>
<hr>
<h3>시험 포인트</h3>
<ul>
<li>AWS 서비스의 온프레미스 확장</li>
<li>단순 VPN 아님</li>
</ul>
<hr>
<h2>AWS Wavelength</h2>
<h3>개요</h3>
<ul>
<li><strong>5G 네트워크 기반 초저지연 컴퓨팅</strong></li>
<li>통신사 네트워크 내부에 AWS 인프라 배치</li>
</ul>
<hr>
<h3>Wavelength 특징</h3>
<ul>
<li>지연 시간 극소화 (ms 단위)</li>
<li>모바일, AR/VR, 자율주행에 적합</li>
<li>EC2 인스턴스를 통신사 엣지에서 실행</li>
</ul>
<hr>
<h3>사용 사례</h3>
<ul>
<li>실시간 게임</li>
<li>AR / VR</li>
<li>IoT</li>
<li>자율주행</li>
</ul>
<hr>
<h3>시험 포인트</h3>
<ul>
<li>5G</li>
<li>초저지연</li>
<li>모바일 중심</li>
</ul>
<hr>
<h2>AWS Local Zones</h2>
<h3>개요</h3>
<ul>
<li><strong>AWS 리전을 사용자 가까운 위치로 확장</strong></li>
<li>특정 도시 단위 서비스 제공</li>
</ul>
<hr>
<h3>Local Zones 특징</h3>
<ul>
<li>리전과 연결된 확장 영역</li>
<li>지연 시간 최소화</li>
<li>EC2, EBS, RDS 등 일부 서비스 제공</li>
</ul>
<hr>
<h3>Wavelength vs Local Zones</h3>
<p>| 항목      | Local Zones   | Wavelength         |
| --------- | ------------- | ------------------ |
| 위치      | AWS 확장 지역 | 통신사 5G 네트워크 |
| 목적      | 낮은 지연     | 초저지연           |
| 주요 대상 | 엔터프라이즈  | 모바일/5G          |</p>
<hr>
<h2>컴퓨팅 서비스 전체 흐름 정리</h2>
<pre><code>EC2
 ├─ Auto Scaling
 ├─ ELB
 ├─ Elastic Beanstalk
 ├─ Lightsail
 ├─ Lambda
 ├─ Batch
 ├─ ECS / EKS / Fargate
 ├─ WorkSpaces / AppStream
 ├─ Outposts
 ├─ Local Zones
 └─ Wavelength
</code></pre>
<hr>
<h2>시험 대비 한 줄 요약</h2>
<ul>
<li><strong>Elastic Beanstalk</strong> → 코드만 올리면 배포</li>
<li><strong>Lightsail</strong> → 초보자용 VPS</li>
<li><strong>WorkSpaces</strong> → 가상 데스크톱</li>
<li><strong>AppStream 2.0</strong> → 앱 스트리밍</li>
<li><strong>Outposts</strong> → AWS 온프레미스</li>
<li><strong>Wavelength</strong> → 5G 초저지연</li>
<li><strong>Local Zones</strong> → 도시 단위 저지연</li>
</ul>
<hr>
<ul>
<li>
<h2>컴퓨팅 서비스 핵심 정리</h2>
<h3>기본 인프라 &#x26; 확장</h3>
<ul>
<li><strong>EC2</strong>
<ul>
<li>AWS의 기본 가상 서버</li>
<li>사용자가 OS·패치·보안 직접 관리</li>
<li>유연하지만 운영 부담 큼</li>
</ul>
</li>
<li><strong>ELB (Elastic Load Balancing)</strong>
<ul>
<li>트래픽을 여러 대상에 자동 분산</li>
<li>장애 인스턴스 자동 제외</li>
<li>고가용성 구성의 핵심</li>
</ul>
</li>
<li><strong>Auto Scaling</strong>
<ul>
<li>트래픽 변화에 따라 EC2 자동 증감</li>
<li>비용 최적화 + 장애 복구</li>
<li>ELB와 함께 사용 시 효과 극대화</li>
</ul>
</li>
</ul>
<p>️ <strong>ELB + Auto Scaling = 안정적·확장형 인프라</strong></p>
<hr>
<h3>애플리케이션 배포 &#x26; 간소화</h3>
<ul>
<li><strong>AWS Elastic Beanstalk</strong>
<ul>
<li>코드만 업로드하면 실행 환경 자동 구성</li>
<li>EC2, ELB, Auto Scaling을 자동 관리</li>
<li>PaaS 개념</li>
<li>서버 접근 가능 (서버리스 아님)</li>
</ul>
</li>
<li><strong>Amazon Lightsail</strong>
<ul>
<li>초보자용 VPS 서비스</li>
<li>간단한 UI + 월 정액 요금</li>
<li>소규모 웹사이트, 개인 프로젝트에 적합</li>
<li>대규모 확장에는 부적합</li>
</ul>
</li>
</ul>
<hr>
<h3>서버리스 컴퓨팅</h3>
<ul>
<li><strong>AWS Lambda</strong>
<ul>
<li>이벤트 기반 서버리스 컴퓨팅</li>
<li>서버 관리 불필요</li>
<li>실행한 만큼만 비용 지불</li>
<li>자동 확장</li>
</ul>
</li>
</ul>
<p>️ <strong>Lambda = 이벤트 기반 서버리스</strong></p>
<hr>
<h3>배치 &#x26; 대규모 연산</h3>
<ul>
<li><strong>AWS Batch</strong>
<ul>
<li>대규모 배치 작업 처리</li>
<li>작업량에 따라 컴퓨팅 자원 자동 프로비저닝</li>
<li>컨테이너 기반 실행</li>
<li>과학 계산, 렌더링, 대량 데이터 처리에 적합</li>
</ul>
</li>
</ul>
<hr>
<h3>컨테이너 &#x26; 마이크로서비스</h3>
<ul>
<li><strong>Amazon ECS</strong>
<ul>
<li>AWS 자체 컨테이너 오케스트레이션</li>
<li>Docker 기반</li>
<li>설정과 운영이 비교적 단순</li>
</ul>
</li>
<li><strong>Amazon EKS</strong>
<ul>
<li>관리형 Kubernetes 서비스</li>
<li>대규모·복잡한 컨테이너 환경에 적합</li>
<li>오픈소스 Kubernetes 사용</li>
</ul>
</li>
<li><strong>AWS Fargate</strong>
<ul>
<li>서버리스 컨테이너 실행 환경</li>
<li>서버·클러스터 관리 불필요</li>
<li>ECS / EKS와 함께 사용</li>
</ul>
</li>
</ul>
<p>️ <strong>ECS / EKS / Fargate = 컨테이너 기반 마이크로서비스</strong></p>
<hr>
<h3>가상 데스크톱 &#x26; 애플리케이션 제공</h3>
<ul>
<li><strong>Amazon WorkSpaces</strong>
<ul>
<li>클라우드 가상 데스크톱(VDI)</li>
<li>재택근무·보안 환경에 적합</li>
<li>전체 데스크톱 제공</li>
</ul>
</li>
<li><strong>Amazon AppStream 2.0</strong>
<ul>
<li>애플리케이션 스트리밍 서비스</li>
<li>데스크톱이 아닌 앱 단위 제공</li>
<li>웹 브라우저로 실행</li>
</ul>
</li>
</ul>
<p>️ 데스크톱 → <strong>WorkSpaces</strong>
️ 앱만 제공 → <strong>AppStream 2.0</strong></p>
<hr>
<h3>하이브리드 &#x26; 엣지 컴퓨팅</h3>
<ul>
<li><strong>AWS Outposts</strong>
<ul>
<li>AWS 인프라를 온프레미스에 설치</li>
<li>동일한 AWS 콘솔로 관리</li>
<li>데이터 로컬 처리 및 규제 대응</li>
</ul>
</li>
<li><strong>AWS Local Zones</strong>
<ul>
<li>특정 도시 단위로 AWS 리전 확장</li>
<li>낮은 지연 시간 제공</li>
<li>엔터프라이즈 워크로드 대상</li>
</ul>
</li>
<li><strong>AWS Wavelength</strong>
<ul>
<li>통신사 5G 네트워크 내부에 AWS 인프라 배치</li>
<li>초저지연(ms 단위)</li>
<li>모바일·AR/VR·실시간 서비스에 적합</li>
</ul>
</li>
</ul>
